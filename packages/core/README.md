# @scalerrs/core

the core library - fetch, parse, check, and generate meta for google doc articles.

## install

```sh
bun add @scalerrs/core
```

## usage

```ts
import { extractFromHtml, runChecks, buildMeta, toMarkdown } from "@scalerrs/core"

const html = await fetch(`https://docs.google.com/document/d/${docId}/export?format=html`).then(r => r.text())

const extracted = extractFromHtml(html)
const markdown = toMarkdown(extracted.bodyHtml)

const [report, meta] = await Promise.all([
  runChecks(extracted),
  buildMeta(markdown, extracted.metas),
])
```

## what it checks

**static**

- single h1, heading hierarchy (no skipped levels)
- meta title 50–60 chars, meta description 120–160 chars
- word count, image count relative to length
- all images have alt text, images are publicly reachable
- no duplicate or broken links

**ai (claude)**

- logical structure, lexical quality, repetition
- sentence length, section transitions
- keyword stuffing, call to action, tone consistency
- product link balance

## meta generation

fields defined in the doc (`Meta Title:`, `Meta Description:`, `Meta Keywords:`) take priority. anything missing is
generated by claude. `sources` tells you which is which.
